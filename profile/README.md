# 🏛️ 한양대학교 기획처 · Office of Planning & Strategy

> 데이터 기반 대학 행정을 위한 AI·디지털 전환 프로젝트를 관리합니다.  
> Managing AI and digital transformation projects for data-driven university administration.

---

## 📌 소개 · About

한양대학교 기획처는 대학의 중장기 발전계획 수립, 예산 편성 및 운용, 성과 평가, 경영 혁신을 담당하는 핵심 행정 부서입니다.

이 GitHub 조직 계정은 기획처 내 **AI 및 디지털 업무 혁신 프로젝트**의 소스코드, 문서, 데이터 분석 결과물을 체계적으로 관리하기 위해 운영됩니다.

> Hanyang University's Office of Planning & Strategy is responsible for long-term strategic planning, budget management, performance evaluation, and administrative innovation.  
> This GitHub organization manages source code, documentation, and data analysis outputs for AI and digital transformation initiatives.

## 🔒 저장소 접근 정책 · Repository Access Policy

```
Public  Repository   — 공개 산출물, 가이드 문서
Private Repository   — 내부 데이터, 개발 중인 프로젝트
```

- **외부 공개(Public)** repo는 민감 정보가 포함되지 않은 산출물·문서에 한합니다.
- **내부(Private)** repo는 팀 단위로 접근 권한이 부여되며, 팀 외 閱覧은 관리자 승인이 필요합니다.
- 개인정보, 예산 원자료 등 민감 데이터는 GitHub에 업로드하지 않습니다.

> Access to private repositories is granted on a per-team basis.  
> Sensitive data such as personal information or raw budget data must not be uploaded to GitHub.

## 🌿 협업 규칙 · Contribution Guidelines

모든 팀원은 아래 규칙을 준수합니다.

1. **`main` 브랜치 직접 push 금지** — 반드시 feature 브랜치에서 작업 후 PR 제출
2. **브랜치 네이밍 규칙 준수**
   ```
   feature/작업내용      예) feature/budget-report-2025
   fix/수정내용          예) fix/dashboard-data-error
   docs/문서내용         예) docs/api-guide-update
   ```
3. **PR 제출 시 리뷰어 반드시 지정**
4. **커밋 메시지는 작업 내용을 명확하게** 작성

> All contributors must follow these collaboration rules.  
> Direct pushes to `main` are prohibited. All changes must go through a Pull Request with at least one reviewer.
